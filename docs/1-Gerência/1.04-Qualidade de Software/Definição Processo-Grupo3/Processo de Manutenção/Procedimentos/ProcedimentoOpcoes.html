2.3 - Procedimento para Desenvolvimento de Opções (PDO)

O PDO busca elucidar opções para implementar a modificação. No mínimo duas opções devem ser desenvolvidas de acordo com a análise para implementar a mudança.

Os passos a serem realizados são:

           & nbsp;    

a) Atribuir uma prioridade de trabalho no Relato de Problema (RP) ou Solicitação de Mudança (SM) – Após a análise, será atribuída uma prioridade para lidar e/ou tratar os RP/RMs.

b) Determinar se existem soluções alternativas para o problema – Significa prover ao usuário ou operador orientações de outras formas de solucionar os problemas relatados (RP).

c) Definir requisitos para a modificação – Para que a modificação seja considerada, é necessário definir um conjunto de requisitos que ela deve cumprir, isto é, deve estar dentro de limites estabelecidos (ser realista, ser testável, estar de acordo com o domínio e magnitude do software em si).

d) Definir o tamanho e magnitude da modificação.

e) Desenvolver diferentes opções para implementação da modificação.

f) Armazenar se as opções propostas foram aceitas ou recusadas – Junto à opção, deve ser documentado se ela foi aprovada ou recusada para implementação. Se uma opção for considerada como recomendada pelo mantenedor, deve-se documentar esta decisão junto à opção (no Documento de Opções de Implementação).
