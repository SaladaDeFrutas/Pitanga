2.2 - Procedimento para Verificação do Problema (PVP)

O PVP é realizado para identificar se os relatórios de problemas são válidos, de modo que a replicação ou verificação do problema seja obtida. Isso significa encontrar a ocorrência do problema que foi relatado (exatamente como relatado) e se possível conseguir reproduzi-la.          Os passos a serem realizados são:

        

a) Desenvolver uma estratégia de teste para verificar o problema – Isso inclui determinar o tipo de teste (estrutural, funcional), os critérios, os casos de teste, os itens a serem testados e porque da escolha destes itens (baseado em risco ou por outro motivo), para que o esforço com os testes seja diminuído (realizar vários testes sem objetividade) e os problemas possam ser identificados ou replicados de uma maneira satisfatória.

b) Obter da Gerência de Configuração a versão do software afetada – Neste passo, a versão na qual o problema foi relatado deve ser recuperada do repositório do projeto corrente (Git) para a Workspace do mantenedor e logo após ser instalada.

c) Rodar os testes para verificar o problema – Os testes serão executados, preferencialmente com uma cópia dos dados afetados, e os resultados são documentados e postos sobre o controle da gerência de configuração (neste momento será ativado o processo de depuração).

d) Realizar a classificação de anomalias, através do problema identificando os defeitos e falhas (instancia-se o processo de classificação de anomalias).

e) Documentar as estratégias de teste – A documentação de teste deve ser atualizada caso necessária, incluindo os planos de teste, os procedimentos de teste e os relatórios.

* O Procedimento de Verificação de Problema só é ativado quando se obtém um relato de um problema.
